# VERDICT — BUA Steam: Non-Steam Launcher Fixes

## Status: TBD (Fix 1 validated, Fix 2 not yet implemented)

## Summary

Non-Steam game launchers generated by `create-steam-launchers.sh` have no sound and broken controller input because they invoke Proton directly, bypassing the Steam Linux Runtime container. The fix is to use the same launch chain Steam itself uses: `SteamLinuxRuntime_4/_v2-entry-point --verb=run -- proton run`. This was validated on-device with eXceed 2nd - Vampire REX (sound + 8BitDo Lite 2 controller both working correctly).

A secondary issue (artwork dimension hardcoding) still needs implementation.

## Plan vs reality

Fix 1 required additional env vars not anticipated in initial research:
- `PROTON_NO_STEAM_OVERLAY=1` alone was insufficient; needed `WINEDLLOVERRIDES="lsteamclient=d;steam.exe=d"` to suppress the Wine steamclient assertion entirely.
- The discovery came from reading Steam's own `compat_log.txt` which showed the SLR launch prefix.

## Root Causes

1. Launcher template calls `"$PROTON_PATH" run "$EXE"` directly, missing the SteamLinuxRuntime container that provides audio and SDL environment
2. `sgdb_get_image()` hardcodes `dimensions=460x215` with no fallback

## Discovery Path

1. Observed no sound + inverted Y-axis on 8BitDo Lite 2 when launching from ES
2. Confirmed game worked correctly from Steam Big Picture (Steam mediates launch)
3. Attempted `steam://rungameid/` URL — failed (RunImage binary doesn't relay to inner Steam)
4. Attempted `-applaunch` — failed ("Game configuration unavailable" for shortcut IDs)
5. Found `compat_log.txt` showing Steam's actual launch prefix includes SteamLinuxRuntime_4
6. Used SLR chain directly — game launched but steamclient assertion appeared
7. Added `WINEDLLOVERRIDES` to disable lsteamclient/steam.exe DLLs — fully working
