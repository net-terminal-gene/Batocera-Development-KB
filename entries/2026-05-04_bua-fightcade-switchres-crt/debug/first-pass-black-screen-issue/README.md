# First pass — Fightcade TEST GAME / Switchres black screen (archived)

**Status:** Closed as a working folder. All numbered debug notes from the first investigation live here. **Second pass** uses `../second-pass-black-screen-issue/`.

**Hardware / OS context (from logs):** Batocera v43 (43acouv), Xorg, Openbox, Fightcade under `/userdata/system/add-ons/fightcade`, Nanao MS929 over DP-1 (15 kHz path).

## Chronology — every major move (implementation + ops)

Order is approximate; overlaps occurred during rapid iteration.

### A. Architecture decisions

1. **Injection point:** `fcade://` handled by `$HOME/bin/xdg-open` (generated by BUA `fightcade.sh`). Route to `switchres_fightcade_wrap.sh` instead of calling `fcade.sh` directly for game URLs.
2. **`fcade.sh` returns immediately** (backgrounds `fcade`). Cannot use `switchres -s -l "...fcade.sh..."`; must use **`switchres … -s -k`** (keep mode) and restore manually after emulator exits.
3. **Restore target:** Snapshot **`PRE_MODE`** / **`PRE_RES`** at wrapper entry (menu timing: 576i, 240p, 480i, etc.), not a hardcoded 640×480.

### B. Wrapper logic iterations (`switchres_fightcade_wrap.template.sh`)

4. **Gating `fightcade_should_use_switchres`:** `switchres` executable, `batocera-resolution getDisplayMode` = `xorg`, `currentResolution` width `< 720`, optional `/userdata/system/configs/fightcade-switchres.force`.
5. **`wait_for_emulators` fix:** Early version polled `pgrep` only after 2s; if Wine had not started `fcadefbneo.exe` yet, the loop skipped and **restore ran during load** → black. Fixed by **wait until emu process appears**, then wait until gone, then settle sleep.
6. **`display_matches_snapshot`:** After Switchres, `batocera-resolution currentMode`/`currentResolution` often empty; added **`xrandr` `current WxH`** fallback for matching.
7. **`restore_display_mode`:** Retries `setMode`, **`xrandr`** from `PRE_MODE`, **`minTomaxResolution`** last resort; removed brittle **`forceMode`** / width≥600 heuristics.
8. **`fightcade_pick_display`:** `DISPLAY` from `/tmp/.X11-unix/X0` or `X1` (avoid hardcoded `:0` after bad SSH/service restarts).
9. **Timing:** Post-`-k` **`sleep`** tuned repeatedly (2→3→4→5→9→back to 4 in repo) for launch black; **9s did not solve persistent black** (UX cost too high).
10. **`poll_snapshot_match`** + faster restore path; later **`EXIT`/`INT`/`TERM`/`HUP` trap** to run restore if wrapper aborts (**`kill -9`** skips bash `EXIT` trap** — documented limitation).
11. **`fightcade_ensure_menu_baseline`:** Before each Switchres, if snapshot looks like stuck arcade geometry (**width `< 512`** and **height `≤ 224`**), **`minTomaxResolution`** then re-snapshot (avoid snapshotting 384×224 as “menu”). Comments once broke parsing on device; parentheses-heavy comment removed (**syntax error** on remote).
12. **`fightcade-switchres.disable`:** If **`/userdata/system/configs/fightcade-switchres.disable`** exists, wrapper **passes through** to `fcade.sh` only (no Switchres) for emergency playability.
13. **Emergency script:** `fightcade_display_recover.sh` (**`minTomax`** then **`641x480i`** fallback); warns **do not** `/etc/init.d/S31emulationstation restart` from SSH (spawned second X on `:1` once).

### C. SSH / recovery moves (when raster stuck)

14. **`batocera-resolution minTomaxResolution`** + **`xrandr --output DP-1 --mode 641x480i --rate 59.98`** to return menu timing when stuck in **384×224**.
15. **`pkill` / `kill -9`** stuck **`switchres_fightcade_wrap.sh`** when hung; killed **`fcadefbneo`** when needed.
16. **Incident:** Remote **`switchres_fightcade_wrap.sh`** line ~271 **syntax error** (`r it).`) — corrupted comment → **`xdg-open` wrapper failed** → Fightcade would not open. Fixed comment in template, **rsync + `sed`** regenerate on device, **`bash -n`** verify.
