# Research — BUA Steam Non-Steam Game Launchers

## Findings

### Live System Validation (2026-03-06)

SSH session to `batocera.local` — read-only inspection of a live Batocera system with BUA Steam installed.

### Dual File Types: `.sh` and `.steam`

The live system has **both** `.sh` and `.steam` files in `/userdata/roms/steam/`:

- **`.sh` files** — Generated by `create-steam-launchers.sh`. Named `APPID_Name.sh`. Full bash scripts that launch games via `steam -gamepadui -silent -applaunch APPID`.
- **`.steam` files** — Contain a single line: `steam://rungameid/APPID`. These are the format that upstream Batocera's ES system config expects (see ES config below).
- **`.sh.keys` files** — Padtokey profiles for `.sh` launchers (hotkey+start → `pkill -f steam`).

The `.steam` files are the **official** ES extension. The `.sh` files work because `batocera.conf` overrides the emulator to `steam.emulator=sh` / `steam.core=sh`.

### ES System Configuration for Steam

From `/usr/share/emulationstation/es_systems.cfg` on the live system:

```
<name>steam</name>
<path>/userdata/roms/steam</path>
<extension>.steam</extension>
<command>emulatorlauncher %CONTROLLERSCONFIG% -system %SYSTEM% -rom %ROM% ...</command>
```

**The official extension is `.steam`, not `.sh`.** The `.sh` files work via the `batocera.conf` override (`steam.emulator=sh`, `steam.core=sh`).

### `.steam` File Format

Each `.steam` file contains a single line — the `steam://rungameid/` URL:

```
steam://rungameid/2379780
```

Examples from the live system:
- `Balatro.steam` → `steam://rungameid/2379780`
- `SILENT HILL f.steam` → `steam://rungameid/2947440`
- `- ASURA THE STRIKER -.steam` → `steam://rungameid/2389830`
- `Steam.steam` → `com.valvesoftware.Steam` (special — launches Steam itself)

### `.sh` Launcher Template (Full — from live system)

```bash
#!/bin/bash
export RIM_ALLOW_ROOT=1
export HOME=/userdata/system/add-ons/steam
ulimit -H -n 819200 && ulimit -S -n 819200
#------------------------------------------------
# Steam Game Launcher
# Game: Balatro
# AppID: 2379780

APPID="2379780"
STEAM_DIR="/userdata/system/add-ons/steam"
STEAM_LAUNCHER="${STEAM_DIR}/steam"

cd "$STEAM_DIR" || exit 1

# Launch game via Steam
"$STEAM_LAUNCHER" -gamepadui -silent -applaunch "$APPID" &
STEAM_PID=$!

# Wait for the game process to finish
wait $STEAM_PID

# Kill all remaining Steam processes when game closes
pkill -f steam 2>/dev/null || true
pkill -f steamwebhelper 2>/dev/null || true
#------------------------------------------------
```

Key: uses `steam -gamepadui -silent -applaunch APPID`, not `steam://rungameid/`.

### `create-steam-launchers.sh` (Full Source — from live system)

Runs as a background loop (every 5 seconds) started by the Launcher. For each `appmanifest_*.acf` in `steamapps/`:

1. Parses `appid`, `name`, `installdir` from the ACF
2. Creates `APPID_Name.sh` (bash launcher) + `.sh.keys` (padtokey)
3. Downloads header image from `cdn.cloudflare.steamstatic.com`
4. Appends entry to `gamelist.xml`

**Only scans `appmanifest_*.acf`** — confirms it never touches `shortcuts.vdf`.

### Launcher Script (from live system)

`/userdata/system/add-ons/steam/Launcher`:
- Sets `DISPLAY=:0.0`, `HOME`, env
- Starts `create-steam-launchers.sh &` in background
- Starts `lbfix.sh &` (lib fix) in background
- Launches `steam -gamepadui` or `steam` based on parameter
- Monitors via `wmctrl` for Steam window
- On Steam close: kills launcher generator, kills remaining steam processes, calls `curl http://127.0.0.1:1234/reloadgames`

### shortcuts.vdf — Live Data (2026-03-06)

**Location (per-user, not global):**
```
/userdata/system/add-ons/steam/.local/share/Steam/userdata/1080337349/config/shortcuts.vdf
```

Note: the initial assumption was that it would be in `Steam/config/shortcuts.vdf` (global). It is actually in `Steam/userdata/<STEAM_USER_ID>/config/shortcuts.vdf` (per-user). The user ID `1080337349` is the Steam account's numeric ID.

**File size:** 348 bytes (one entry)

**Test game added:** Maldita Castilla (extracted from `/userdata/roms/windows/Maldita Castilla.wsquashfs` to `/userdata/system/add-ons/steam/MalditaCastilla/`)

**Parsed fields from hex dump:**

| Field | Value |
|-------|-------|
| `appid` (raw LE bytes) | `d2 25 d9 df` → `0xDFD925D2` → **3755861458** |
| `AppName` | `Maldita Castilla.exe` |
| `Exe` | `"/root/MalditaCastilla/Maldita Castilla.exe"` |
| `StartDir` | `/root/MalditaCastilla/` |
| `icon` | (empty) |
| `ShortcutPath` | (empty) |
| `LaunchOptions` | (empty) |
| `IsHidden` | 0 |
| `AllowDesktopConfig` | 1 |
| `AllowOverlay` | 1 |
| `OpenVR` | 0 |
| `Devkit` | 0 |
| `LastPlayTime` | 0 |
| `FlatpakAppID` | (empty) |

**Binary format observations:**
- File starts with `\x00shortcuts\x00` header
- Each entry starts with `\x00<index>\x00` (e.g. `\x000\x00`)
- Field types: `\x02` = uint32, `\x01` = string (null-terminated)
- `appid` is a `\x02` (uint32) field in little-endian
- Strings are null-terminated
- `Exe` path is wrapped in double quotes
- Entry ends with `\x08\x08\x08\x08` terminators

**Key confirmation:** The shortcut ID (3755861458 / 0xDFD925D2) has high bits set, confirming non-Steam IDs are in a separate range from real Steam App IDs.

### Key Insight: `.steam` Files Are the Simpler Path

Since `.steam` files are just `steam://rungameid/APPID`, creating a non-Steam game launcher is as simple as:

1. Get the shortcut ID from `shortcuts.vdf` (or compute it)
2. Create a file: `GameName.steam` containing `steam://rungameid/SHORTCUTID`
3. ES will pick it up via the standard `.steam` extension

**No need to create `.sh` launchers for non-Steam games** — the `.steam` format already supports arbitrary `steam://rungameid/` URLs. The only question is whether `steam://rungameid/` works with non-Steam shortcut IDs (which have high bits set, e.g. `>2000000000`).

### batocera.conf Steam Settings

```
steam.emulator=sh
steam.core=sh
```

Per-game videomode entries reference `.sh` files only:
```
steam["2379780_Balatro.sh"].videomode=854x480.60.00045
```

[Inference] If a non-Steam game is added as a `.steam` file (not `.sh`), it may not be affected by the `steam.emulator=sh` override. This needs testing — the `.steam` files may use the default `emulatorlauncher` path which handles `steam://rungameid/` natively.

### Live Test Session (2026-03-06 / 2026-03-07)

#### Test Game: Maldita Castilla

**Source:** Extracted from `/userdata/roms/windows/Maldita Castilla.wsquashfs` (19MB, smallest wsquashfs with game exe at root).

**Extraction:** `unsquashfs -d /userdata/system/non-steam-test/MalditaCastilla '/userdata/roms/windows/Maldita Castilla.wsquashfs'`

**Steam file browser path issue:** Steam's "Pick game to add" dialog uses `HOME=/userdata/system/add-ons/steam` (not `/root`). `/userdata` is not visible in the root filesystem browser (mergerFS setup). Symlink to `/userdata/system/add-ons/steam/MalditaCastilla` failed — Steam showed symlink as 47-byte file, not folder. **Fix:** Copied the extracted directory into Steam's HOME: `cp -r .../MalditaCastilla /userdata/system/add-ons/steam/MalditaCastilla`

**shortcuts.vdf path:** Per-user at `Steam/userdata/1080337349/config/shortcuts.vdf` (not global `Steam/config/`).

**Shortcut ID extracted:** 3755861458 (0xDFD925D2, little-endian bytes `d2 25 d9 df`).

#### Visibility Test

| Format | File | ES shows it? |
|--------|------|--------------|
| `.steam` | `Maldita Castilla.steam` containing `steam://rungameid/3755861458` | **No** |
| `.sh` | `3755861458_Maldita_Castilla.sh` | **Yes** |

**Conclusion:** With `steam.emulator=sh`, ES only discovers `.sh` files. `.steam` files are ignored.

#### Launch Attempts

**Attempt 1 — `steam://rungameid/3755861458`:** Steam console log shows `Startup - Steam Client launched with: ... 'steam://rungameid/3755861458'`. Steam Big Picture opened. Game did not launch. User reported "did not launch."

**Attempt 2 — `-applaunch 3755861458`:** Updated launcher to match real-game format. Steam console log shows `Startup - Steam Client launched with: ... '-applaunch' '3755861458'`. Steam Big Picture opened. Game did not launch. User reported "stuck."

#### Stuck State Analysis (2026-03-07)

**Process tree:** Launcher script (`3755861458_Maldita_Castilla.sh`) runs `steam -gamepadui -silent -applaunch 3755861458 &` then `wait $STEAM_PID`. Steam process spawns dwarfs mount, bwrap sandbox, Steam client, steamwebhelper. Steam stays running (Big Picture UI). The `wait` never returns because the Steam process tree remains alive.

**Result:** ES shows "launching" indefinitely. User must `pkill -f steam` to return to ES.

**Unresolved:** Neither `steam://rungameid/` nor `-applaunch` successfully launched the non-Steam game. Steam receives both; Big Picture opens; game executable does not start. Possible causes: (a) `-applaunch` may not support shortcut IDs, (b) `steam://rungameid/` may need different handling, (c) path resolution inside Steam's sandbox (`/root/MalditaCastilla` → `/userdata/system/add-ons/steam/MalditaCastilla`) may fail, (d) Proton/compat layer not triggering for shortcut.

#### Launcher Script Transfer

SSH `expect` wrapper mangles `$` in heredocs. **Workaround:** Base64-encode script locally, pipe to remote: `echo $B64 | base64 -d > /userdata/roms/steam/3755861458_Maldita_Castilla.sh`

### Resolved Unknowns

1. ~~`shortcuts.vdf` does not exist yet~~ — **Resolved:** Exists at per-user path after adding non-Steam game
2. ~~`steam://rungameid/SHORTCUTID` with non-Steam IDs~~ — **Resolved: Does not work.** Steam receives it; Big Picture opens; game never starts. Not a viable launch method.
3. ~~`-applaunch SHORTCUTID`~~ — **Resolved: Does not work.** Same behavior as #2. `-applaunch` does not support shortcut IDs.
4. ~~`.steam` vs `.sh` emulator path~~ — **Resolved:** With `steam.emulator=sh`, only `.sh` files appear in ES
5. ~~Launcher wait behavior~~ — **Resolved:** Proton direct launch uses `wait $PROTON_PID` which returns when the game exits (no persistent Steam process tree)
6. **Working approach: Proton direct launch** — `proton run exe` with `STEAM_COMPAT_DATA_PATH` + `STEAM_COMPAT_CLIENT_INSTALL_PATH`. Verified working 2026-03-07.
