# VERDICT - CRT vertical autoconfig script

## Status: TBD (implementation not started in Batocera-CRT-Script)

## Summary

**Research complete** for vanilla vertical presets: **PCE / PCE CD**, **SNES (snes9x)**, **Saturn (beetle-saturn)**, **Dreamcast (flycast)**, **NAOMI (flycast — same core as Dreamcast)**, **PSX (`pcsx_rearmed`)**, **PSP (`ppsspp`)**, **PS2 (standalone PCSX2-Qt)**, **Switch (citron-emu AppImage)**, **FBNeo + Neo Geo**, and **Vectrex (vecx)**. Each has a dedicated **`research/*-vertical-autoconfig.md`** spec. **Seven recipe classes** identified: (a) **geometry-class** (PCE / SNES / Vectrex / FBNeo) — videomode + ratio + RA keys; (b) **rotation-only** (NAOMI) — one RA rotation key; (c) **state-injection** (Saturn / Dreamcast) — RA savestates + decoupled autoload keys; (d) **rotation + fill + per-game custom viewport** (PSX); (e) **state-injection + two-tier rotation + per-game viewport + TATE input remap** (PSP); (f) **standalone-emulator bootstrap-state w/ file-lock** (PS2); (g) **standalone Citron AppImage + in-game TATE save** (Switch). Saturn captures TATE inside Saturn RAM (state file is enough); Dreamcast also needs per-game `Flycast/<stem>.cfg` with `video_rotation = "0"` for in-game-TATE titles. PSP adds locked D-pad remaps for native-TATE titles (PPSSPP libretro rotates picture but not pad) and splits horizontal-on-vertical titles into viewport-only tier. PS2 uses PCSX2-Qt `-statefile` instead of RA autoload. Switch uses unofficial add-on `citron-emu` + operator Options-menu TATE (persists to `yuzu/sdmc`); no RA layer. State-injection libretro systems use **decoupled autoload-only** RA keys (`<system>.autosave=0` + explicit `..savestate_auto_load=true` + `..savestate_auto_save=false`). Shell generator in CRT Script is still **TBD**.
