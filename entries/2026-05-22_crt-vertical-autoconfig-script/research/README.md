# Research - CRT vertical autoconfig script

## Purpose of this directory

**`research/` holds the implementation spec:** recommended file paths, `batocera.conf` keys, RetroArch option names, and ordering constraints for the vertical autoconfig **before** code lands in Batocera-CRT-Script. Treat findings here as the checklist the shell generator must satisfy.

Sub-files:

| [fbneo-vertical-autoconfig.md](fbneo-vertical-autoconfig.md) | FBNeo + Neo Geo, `FinalBurn Neo/` config dir, rotation policy |
| [vectrex-vertical-autoconfig.md](vectrex-vertical-autoconfig.md) | **`vectrex.*`** `batocera.conf` merge, `crt_switch_resolution` off, `video_rotation`, `ratio=full`, optional manifest |
| [snes-vertical-autoconfig.md](snes-vertical-autoconfig.md) | **`snes.*`** merge: **`256x256.*`** preferred for `full` on vertical cab, **`crt_switch` off**, **`crop_overscan` false**, **`snes9x_gfx_clip` off**, optional manifest |
| [saturn-vertical-autoconfig.md](saturn-vertical-autoconfig.md) | **State-injection** spec (not geometry): decoupled autoload-only (`saturn.autosave=0` + `saturn.retroarch.savestate_auto_load=true` + `..._save=false`) + bundled `*.state.auto` files; v43 geometry untouched; `beetle-saturn` core lock |

Default `cave_family: [DoDonPachi, Donpachi]`. `pcsx_rearmed` core lock (memory cards are core-independent but cfgs are core-tied) |

## Findings (high level)

- Vanilla vertical cabinet already validated **global** `pcengine` + `pcenginecd` `videomode` + `ratio` (see linked PCE doc under vanilla portable session).
- FBNeo on vanilla may use **`fbneo.video_allow_rotate=off`** at system level; per-ROM `.cfg` / `.opt` remain title-specific.
- **Vectrex (vecx):** validated **`vectrex.videomode`** + **`vectrex.ratio=full`** + **`vectrex.retroarch.crt_switch_resolution=0`** + **`vectrex.retroarch.video_rotation=3`**; ES restart required after edits (see [vectrex-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/vectrex-vertical-vanilla-v43.md)).
- **SNES:** validated **`snes.videomode=256x256.60.00006`**, **`snes.ratio=full`**, **`snes.retroarch.crt_switch_resolution=0`**, **`video_crop_overscan=false`**, **`snes9x_gfx_clip` off**; **`256x448.*`** rejected on reference cab; ES restart after edits (see [snes-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/snes-vertical-vanilla-v43.md)).
- **Saturn (Beetle Saturn):** v43 geometry already correct on this cabinet; **state-injection** path used instead — decoupled autoload-only keys (`saturn.autosave=0` + `saturn.retroarch.savestate_auto_load=true` + `..._save=false`) plus 38 `.state.auto` files seeded from Myzar (`10.23.6.211`) to v43 (`10.23.6.210`). Behavior: skip Sega/Saturn/intro splashes, in-game TATE / Side preserved via Saturn RAM in the savestate; pristine baseline survives every quit. Myzar's `Beetle Saturn/*.cfg` geometry rejected (wrong on this cabinet). See [saturn-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/saturn-vertical-vanilla-v43.md).
- **Dreamcast (Flycast):** cabinet-deployed 2026-05-22 with 18 vertical-shmup titles, full per-title rotation matrix validated. Same decoupled autoload-only state pattern as Saturn, **plus** system-wide `dreamcast.retroarch.video_rotation=3` (writes into `retroarchcustom.cfg` at launch, immune to RA per-content override races), **plus** per-game `Flycast/<stem>.cfg` with `video_rotation = "0"` for 9 in-game-TATE titles (Ikaruga, Karous, NEO XYX, Triggerheart Exelica, Trizeal, Under Defeat, Radirgy, Psyvariar 2, Shikigami no Shiro II — these games' Myzar savestates captured them with the in-game TATE menu on, so RA rotation would double-rotate), **plus** `dreamcast.ratio=full` + `dreamcast.retroarch.video_force_aspect=false` for fill geometry. Myzar's `Flycast/dreamcast.cfg` geometry rejected (wrong polarity + wrong viewport for this cab; Myzar uses `reicast_screen_rotation=vertical` core option which we did not adopt). See [dreamcast-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/dreamcast-vertical-vanilla-v43.md) for the full per-title rotation matrix.
- **NAOMI (Flycast, same core as Dreamcast):** cabinet-deployed 2026-05-22 with one verified title (Ikaruga), 11 expected-PASS titles pending per-title sweep. **One key only:** `naomi.retroarch.video_rotation=3`. No state-injection (arcade ROMs have no skippable splash), no fill keys (640×480 native fills rotated super-res), no autosave decoupling. NVRAM service-mode TATE (`/userdata/saves/naomi/reicast/<rom>.zip.nvmem`) is auto-created on first launch defaulting to TATE off, which is why no per-game overrides are needed on this cabinet's clean install. Per-game `Flycast/<stem>.cfg` `video_rotation = "0"` documented as the fallback for any NVRAM-TATE-on title (mirroring Dreamcast in-game-TATE pattern). Shares `Flycast/` dir with Dreamcast; no stem collision (NAOMI uses MAME short names like `trizeal.cfg`, Dreamcast uses display names like `Trizeal.cfg`). Myzar's `Flycast/naomi.opt` `reicast_screen_rotation=vertical` + custom 640×960 viewport rejected for the same reasons as Dreamcast. See [naomi-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/naomi-vertical-vanilla-v43.md) for the per-title status table.
- **PSX (`pcsx_rearmed`):** cabinet-deployed 2026-05-23 with 10 of 30 roster titles verified — 8 via system-wide keys only, 2 (Cave family: DoDonPachi, Donpachi) via per-game custom-viewport cfg + in-game TATE enabled via the game's Options menu (persisted to the PSX virtual memory card `/userdata/saves/psx/<stem>.1.mcr`). **Fourth recipe class: "Rotation + fill + per-game custom viewport"** — three `psx.*` keys (`retroarch.video_rotation=3` + `ratio=full` + `retroarch.video_force_aspect=false`) cover most of the roster; per-game `PCSX-ReARMed/<stem>.cfg` with full custom viewport (`aspect_ratio_index="24"` + `custom_viewport_width="480"`/`height="640"` + `video_rotation="0"` + `video_scale_integer="false"`) handles Cave-family YOKO-with-side-panels titles. No state-injection (PSX persists in-game TATE to its own memory card; no skippable Sega-style splash to bypass). Memory cards are core-independent across `pcsx_rearmed` / `mednafen_psx` / `swanstation` / `duckstation`, but per-game cfgs are core-tied (dir name `PCSX-ReARMed/` only valid for `pcsx_rearmed`). Myzar uses the same custom-viewport pattern in `Beetle PSX HW/<stem>.cfg` (different core dir name, same recipe shape). PSX BIOS in `/userdata/bios/` required. See [psx-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/psx-vertical-vanilla-v43.md) for the per-title status table and per-game cfg generator commands.
- **PSP (libretro ppsspp):** cabinet-deployed 2026-05-24 with all 6 roster titles validated on v43 `10.23.6.210`. **New sixth recipe class: "State-injection + two-tier rotation + per-game viewport + TATE input remap"** — ten `psp.*` system-wide keys + core `PPSSPP.cfg` + two title tiers: TATE (Star Soldier, Beta Bloc, Neo Geo Heroes Ultimate Shooting) get 480×640 viewport, `video_rotation=0`, locked cfg+rmp with 90° CW D-pad remap (v43 addition beyond Myzar), Myzar `.state.auto` for skip-menu autoload; horizontal (Pac-Man CE, Super Stardust, Space Invaders) get viewport-only cfgs at 544×480 or 816×480 with rotation 3 and no input remap. **Space Invaders exception:** Myzar `.state.auto` removed after it hung RetroArch on v43 (PPSSPP core differs v41→v43); per-game `savestate_auto_load=false`. ES PSP menu clobbers system keys. See [psp-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/psp-vertical-vanilla-v43.md).
- **Switch (Citron AppImage via unofficial add-on):** cabinet-deployed 2026-05-24 with all 3 Cave roster titles validated on v43 `10.23.6.210` (DoDonPachi Resurrection, Espgaluda 2, Mushihimesama). **New seventh recipe class: "Standalone Citron AppImage + system keys + in-game TATE save"** — seven `switch.*` keys lock `citron-emu`, `864x486.60.00070`, Vulkan, stretch (`yuzu_ratio=5`), 1× scale; operator enables in-game TATE once per title (persists to `yuzu/sdmc`). No RetroArch, no state-injection, no per-game batocera keys. Stock `switch.emulator=citron` rejected (missing binary). See [switch-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/switch-vertical-vanilla-v43.md).
- **PS2 (standalone PCSX2-Qt):** cabinet-deployed 2026-05-24 with 12 of 13 vertical-shmup roster titles wired and verified (Triggerheart Exelica Enhanced SLPM-55052 dropped — BIOS region loop on European SCPH30004R; DC version preferred). **New fifth recipe class: "Standalone-emulator bootstrap-state w/ file-lock"** — first standalone-emulator recipe in this project. All prior recipes target libretro via RetroArch cfg files; PS2 has no `.cfg` mechanism, every setting flows through `batocera.conf` and gets materialized into `PCSX2.ini` + pcsx2-qt CLI args by `pcsx2Generator.py`. Nine `ps2.*` system-wide keys (`emulator=pcsx2` + `autosave=0` + `incrementalsavestates=0` + `pcsx2_bilinear_filtering=0` + `pcsx2_blur=true` + `pcsx2_gfxbackend=14` Vulkan + `pcsx2_vsync=1` + `pcsx2_resolution=1` + `pcsx2_texture_filtering=2`) cover all titles uniformly — no per-game render overrides. Per-title `ps2["<ROM>"].state_filename` + `state_slot=1` + `chmod 444` on operator-captured `.01.p2s` bootstrap → configgen appends `-statefile <path> -stateindex 1` to pcsx2-qt command → auto-load before frame 1 → cabinet boots directly into TATE gameplay. **Hard lock: libretro PS2 (`pcsx2_libretro`) is broken on BC250** (AVX-512 SIGILL); even if a future v43 build ships a non-AVX-512 libretro core, the per-game CLI-args mechanism is standalone-only. **Configgen bug:** `ps2.pcsx2_fastboot=true` writes `EnableFastBoot=false` due to inverted `return_values=("false","true")` in `pcsx2Generator.py` line 273; cosmetic only because `-statefile` suppresses BIOS animation regardless. **Optional bundling:** `.01.p2s` bootstraps (~10–15 MB each) tied to specific BIOS version embedded in save state header; default `bundle_bootstraps: no` (operator F1 ritual is 2 minutes per title). See [ps2-vertical-vanilla-v43.md](../../2026-05-20_crt-vanilla-vertical-portable/research/ps2-vertical-vanilla-v43.md) for the per-title status table and bootstrap workflow.
- Myzar **Sai-only** `xrandr` + Switchres coexistence is **not** a default for this script; cite `2026-05-21_crt-myzar-dp-hybrid-switchres` only when documenting optional advanced recipes.
